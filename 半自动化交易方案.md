## 1. 交易频率选择

### 结论：建议使用 **日线级别**，收盘后分析，第二天操作

### 各频率对比

| 维度 | 日线（推荐✅） | 周线 | 盘中/分钟级 |
|------|:-----------:|:----:|:----------:|
| **操作频率** | 每天收盘后跑一次 | 每周五跑一次 | 持续运行 |
| **数据获取** | 免费AKShare足够 | 免费AKShare足够 | 需要实时行情 |
| **时间投入** | 每天10-20分钟 | 每周30分钟 | 需要持续盯盘 |
| **交易成本** | 低（月换手1-4次） | 极低 | 高（可能频繁交易） |
| **学习反馈** | 快（每天有结果） | 慢（每周才一次） | 太快（噪音多） |
| **T+1适配** | 完美匹配 | 匹配 | 受限 |
| **心理压力** | 适中 | 低 | 高 |
| **适合策略** | 趋势/动量/均值回归 | 因子选股/资产配置 | 日内波段（不推荐新手） |

### 为什么日线最优？

```
1. T+1制度天然匹配 —— 今天收盘后分析，明天开盘买入，收盘后才能卖
   你无法日内高频操作，日线是A股最自然的节奏。

2. 反馈周期适中 —— 周线太慢，等三个月才能验证策略；
   日线每天都有反馈，一个月就有20+笔样本。

3. 数据免费且充足 —— AKShare等免费数据源日线数据质量高，
   分钟线和Tick数据要么收费，要么质量堪忧。

4. 时间投入可控 —— 每天收盘后（15:00）花10-20分钟：
   - 跑脚本（1分钟自动）
   - 看信号（5分钟）
   - 决定是否跟单（5分钟）
   - 第二天盘中花几分钟下单

5. 策略不易过拟合 —— 日线数据量适中（10年约2500根K线），
   既不会数据太少统计不显著，也不会数据太多诱人过拟合。
```

### 日常操作节奏（典型的一天）

```
15:00 收盘
15:05 自动运行脚本（定时触发）
       → 拉取今日行情数据
       → 更新本地数据库
       → 运行策略逻辑
       → 生成明日交易信号
       → 推送到手机（微信/钉钉/Bark）
15:10 你在手机上收到信号
       → 查看推荐买卖列表
       → 结合自己的判断，决定是否执行
       → 做一个简单的记录

第二天
09:25 集合竞价期间
       → 确认你要下单的股票
09:30 开盘
       → 手动在券商APP下单
       → 设置止损提醒（如果有）

循环往复
```

### 方案一：在线量化平台（聚宽/米筐）

#### 工作流

```
登录聚宽 → 写策略代码 → 平台自带数据 → 在线回测 → 每日生成信号
                                                      ↓
                                          人工查看信号 → 券商APP下单
```

#### 优点

- **零环境搭建**：浏览器打开就能写策略，不用装Python、不用配环境
- **数据免费**：聚宽自带10+年日线数据、财务数据，无需额外获取
- **回测方便**：写完直接回测，出报告，看曲线，改参数立刻看结果
- **社区丰富**：大量策略模板可直接参考和修改
- **学习曲线最平**：1天内就能跑出第一个回测结果

#### 缺点

- **灵活度受限**：只能在平台框架内写策略，一些自定义逻辑不好实现
- **策略代码在云端**：代码不掌握在自己手里，平台可能关闭或收费
- **信号不自动推送**：需要登录网站查看，没有手机通知
- **数据不归你**：数据在平台，无法导出做深度分析
- **无法对接本地工具**：比如不能和本地数据库、自定义图表联动

#### 适合谁

- 完全零基础，连Python都没装过
- 想快速验证几个策略想法
- 暂时不想折腾技术环境
- 策略逻辑简单（均线、因子选股等标准策略）

---

### 方案二：本地Python脚本（完全自建）

#### 工作流

```
本地Python脚本
├── 每日定时拉取AKShare数据 → 存SQLite
├── 运行策略 → 生成信号
├── 推送到手机（Bark/钉钉/微信）
└── 数据可视化看板（Streamlit，可选）

人工查看信号 → 券商APP下单
```

#### 优点

- **完全控制**：代码、数据、逻辑全在本地，不受平台限制
- **无限扩展**：想加什么数据源、什么策略、什么通知方式都可以
- **数据资产积累**：数据长期存储在自己手里，越来越值钱
- **学习价值最高**：从头搭建，每个环节都深刻理解
- **可对接任何系统**：未来接实盘API、接其他平台都方便
- **完全免费**：AKShare数据免费，所有工具开源

#### 缺点

- **需要搭建环境**：装Python、装库、配定时任务，对新手有门槛
- **初期投入时间多**：搭框架可能需要1-2周
- **需要学基础Python**：至少要能读写修改代码
- **回测需要自己写或接入Backtrader**：比聚宽一键回测复杂

#### 适合谁

- 愿意投入时间学技术
- 长期目标是全自动实盘
- 想拥有自己完整的量化系统
- 策略想法多样化，不想被平台限制

---

### 方案三：混合方案（推荐✅）

#### 工作流

```
阶段A（策略研究，用聚宽）
  聚宽在线回测 → 快速验证策略想法 → 找到可行的策略逻辑
  
阶段B（每日运行，用本地Python）
  本地Python脚本
  ├── AKShare获取日线数据
  ├── 复现聚宽上的策略逻辑
  ├── 生成每日交易信号
  └── Bark/钉钉推送到手机
  
阶段C（信号执行，人工）
  手机收到信号 → 人工判断 → 券商APP手动下单
```

#### 为什么混合方案最好？

```
1. 聚宽用于"探索" —— 快速试错，一天能回测10个策略想法
   不用在本地回测框架上花时间调试

2. 本地脚本用于"执行" —— 简单、可控、能推送通知
   你不需要本地回测引擎，只需要"跑信号"的能力

3. 两个系统独立 —— 聚宽负责验证策略好不好
   本地脚本负责每天告诉你买卖什么

4. 渐进式演进 —— 先跑通最小闭环，再逐步丰富
   不需要一开始就搭完整系统
```

#### 各阶段对应关系

| 活动 | 在哪里做 | 频率 |
|------|----------|------|
| **策略研究/回测** | 聚宽（在线） | 研究阶段集中做 |
| **数据获取** | 本地Python + AKShare | 每日自动 |
| **信号生成** | 本地Python | 每日收盘后 |
| **信号通知** | 手机（Bark/钉钉） | 每日收盘后 |
| **下单执行** | 券商APP手动 | 第二天盘中 |
| **盈亏跟踪** | 本地Python / Excel | 每周/每月 |

## 4. 系统架构设计

### 4.1 数据流详解

```
第1步：数据获取 (15:05自动触发)

  AKShare API
  ├── stock_zh_a_hist()     获取全A股日线数据
  │   参数：每日增量下载（只拉最新几天）
  │   存储：SQLite → daily_kline 表
  │
  ├── stock_zh_a_spot_em()  获取实时股票列表
  │   用途：筛选股票池（排除ST、排除新股等）
  │
  └── stock_board_industry_*  获取行业分类
      用途：行业中性化、行业轮动策略


第2步：数据清洗

  原始数据 → 清洗流水线：
  ├── 复权处理：统一用前复权价格
  ├── 停牌标记：停牌日标记，跳过信号生成
  ├── 异常检测：单日涨跌超过15%标记（可能是数据错误）
  └── 缺失填充：少量缺失用前值填充，大量缺失标记


第3步：策略信号生成

  遍历所有启用的策略：
  
  策略1: 双均线趋势
    输入：股票日线数据
    逻辑：MA(10)上穿MA(30) → 买入信号
          MA(10)下穿MA(30) → 卖出信号
    输出：[(股票代码, 信号类型, 强度, 建议仓位)]

  策略2: 动量突破（同期运行）
    输入：股票日线数据
    逻辑：价格突破20日最高价 → 买入
          跌破20日最低价 → 卖出
    输出：[(股票代码, 信号类型, 强度, 建议仓位)]

  ... 更多策略


第4步：信号汇总

  多策略信号 → 汇总处理：
  ├── 同一股票多个买入信号 → 增强置信度（信号强度叠加）
  ├── 同一股票买卖冲突 → 标记冲突，人工裁决
  ├── 按信号强度排序 → 优先展示强信号
  └── 限制推荐数量 → 最多展示10条买入建议


第5步：风控过滤（三道防线）⭐

  防线一：大盘择时 → 决定能不能买
  ├── 计算大盘（沪深300/上证指数）的均线状态
  ├── 强势（指数>MA20 且 MA20>MA60）→ 正常开仓，仓位系数 1.0
  ├── 震荡（指数>MA20 但 MA20<MA60）→ 谨慎开仓，仓位系数 0.3
  ├── 弱势（指数<MA20 且 MA20<MA60）→ 禁止开新仓，只处理卖出
  └── 极弱（指数<MA60 且持续下跌）→ 建议清仓观望

  防线二：基础规则过滤 → 去掉不能买的
  ├── 排除ST/*ST股票（基本面风险）
  ├── 排除涨停股（买不到）
  ├── 排除跌停股（流动性冻结）
  ├── 排除上市不足60天的新股（数据不足，波动大）
  └── 排除当日停牌股

  防线三：仓位上限控制 → 控制买多少
  ├── 单票仓位上限：不超过总资金 15%（小资金）~ 20%
  ├── 总仓位上限：强势市 ≤80%，震荡市 ≤40%，弱势市 ≤20%
  ├── 单行业上限：不超过总资金 30%
  ├── 买入数量限制：最多同时持有 10 只股票
  └── 最终建议仓位 = 策略建议仓位 × 大盘仓位系数

  详细设计见 [第5节：风控体系](#5-风控体系三道防线)


第6步：通知推送

  最终信号 → 格式化 → 推送到手机
```

### 5.1 防线一：大盘择时

#### 为什么大盘择时是防线的第一关？

```
A股的核心规律：
  - 牛市（趋势向上）：80%的股票上涨，策略信号有效性高
  - 震荡市：策略信号真假参半，频繁打脸
  - 熊市（趋势向下）：80%的股票下跌，再好的策略也难赚钱

不做择时：
  策略在熊市中依然发出买入信号 → 反复止损 → 账户持续缩水

做简单择时：
  熊市自动暂停开仓 → 保存本金 → 等市场转好再出手
```

#### 四档市场状态判断


#### 代码逻辑示意

```python
def get_market_regime(index_df: pd.DataFrame) -> dict:
    """
    判断当前大盘状态
    输入：沪深300指数日线DataFrame
    输出：{'regime': 'strong', 'position_ratio': 1.0, 'label': '强势'}
    """
    close = index_df['close']
    ma20 = close.rolling(20).mean().iloc[-1]
    ma60 = close.rolling(60).mean().iloc[-1]
    latest = close.iloc[-1]
    
    above_ma20 = latest > ma20
    ma20_above_ma60 = ma20 > ma60
    
    # 判断连续下跌
    consecutive_down = 0
    for i in range(len(close)-1, max(0, len(close)-11), -1):
        if close.iloc[i] < close.iloc[i-1]:
            consecutive_down += 1
        else:
            break
    
    if above_ma20 and ma20_above_ma60:
        return {'regime': 'strong',  'position_ratio': 1.0, 'label': '🟢 强势'}
    elif above_ma20 and not ma20_above_ma60:
        return {'regime': 'shaky',   'position_ratio': 0.3, 'label': '🟡 震荡'}
    elif not above_ma20 and not ma20_above_ma60:
        if consecutive_down >= 10:
            return {'regime': 'crash', 'position_ratio': 0.0, 'label': '🔴 极弱'}
        return {'regime': 'weak',    'position_ratio': 0.0, 'label': '🟠 弱势'}
    else:
        # 均线纠缠，偏谨慎
        return {'regime': 'shaky',   'position_ratio': 0.3, 'label': '🟡 震荡'}
```

#### 设计要点

```
1. 为什么用沪深300？
   - 代表A股核心资产
   - 与大部分策略的选股池（沪深300成分股）一致
   - 比上证指数更能反映真实市场状态

2. 为什么用MA20和MA60？
   - MA20（约一个月）：短期趋势
   - MA60（约一个季度）：中期趋势
   - 两个周期结合，能过滤掉大部分震荡噪声
   - 参数少（只有2个），不易过拟合

3. 择时不是万能的：
   - 趋势转折点会有滞后（均线的天然缺陷）
   - 可能错过熊市末期的反弹
   - 但长期来看，躲过的亏损远大于错过的收益
```

### 5.2 防线二：基础规则过滤

这层是"底线规则"——不是用来赚钱的，是用来避免犯低级错误的。

```
│                  基础规则过滤清单                  │
├─────────────┬──────────┬─────────────────────────┤
│ 规则         │ 动作     │ 原因                    │
├─────────────┼──────────┼─────────────────────────┤
│ ST/*ST股票   │ 🔴 排除   │ 基本面极差，退市风险    │
│ 涨停股       │ 🔴 排除   │ 买不到（封板）          │
│ 跌停股       │ 🔴 排除   │ 流动性风险，次日可能继跌 │
│ 上市<60天    │ 🔴 排除   │ 无历史数据，波动巨大    │
│ 停牌股       │ 🔴 排除   │ 无法交易               │
│ 股价<2元     │ 🟡 警告   │ 低价股风险高，标记提醒  │
│ 日均成交<2000万│ 🟡 警告  │ 流动性差，买卖困难      │
```

#### 实现逻辑

```python
def basic_filter(signals: list, stock_info_df: pd.DataFrame) -> list:
    """
    基础规则过滤
    输入：策略信号列表 + 股票信息表
    输出：过滤后的信号列表，排除的会记录原因
    """
    filtered = []
    for sig in signals:
        code = sig['stock_code']
        
        # 获取该股票当日信息
        info = stock_info_df[stock_info_df['code'] == code].iloc[-1]
        
        if info['is_st']:
            sig['filtered'] = True
            sig['filter_reason'] = 'ST股票'
            continue
        if info['pct_change'] >= 9.9:  # 涨停
            sig['filtered'] = True
            sig['filter_reason'] = '涨停，无法买入'
            continue
        if info['pct_change'] <= -9.9:  # 跌停
            sig['filtered'] = True
            sig['filter_reason'] = '跌停，流动性风险'
            continue
        if info['days_since_ipo'] < 60:
            sig['filtered'] = True
            sig['filter_reason'] = '新股，数据不足'
            continue
        if info['suspended']:
            sig['filtered'] = True
            sig['filter_reason'] = '停牌'
            continue
        if info['close'] < 2.0:
            sig['warning'] = '⚠️ 低价股，风险较高'
        if info['amount'] < 20_000_000:  # 2000万
            sig['warning'] = '⚠️ 流动性偏低'
        
        filtered.append(sig)
    
    return filtered
```

### 5.3 防线三：仓位控制

仓位控制决定了**单笔亏损的上限**。

但有个前提必须正视：**仓位策略必须适配资金规模**。A股一手100股的最小交易单位，意味着小资金天然无法分散。

---

#### 5.3.1 先算一笔账：小资金面临的现实约束

```
A股最小交易单位 = 1手 = 100股

不同股价下，买1手需要的资金：
  股价 5元  → 500元    （低价股，如银行股）
  股价 15元 → 1,500元   （中价股）
  股价 50元 → 5,000元   （中高价）
  股价 100元→ 10,000元  （高价股，如茅台一手要17万）

1万元本金能买什么？
  ├── 5元股票：最多买 20手（满仓1只）
  ├── 15元股票：最多买 6手（满仓1只）
  ├── 50元股票：最多买 2手（满仓1只）
  └── 100元股票：根本买不起一手！
```

结论很简单：**1万元根本做不到教科书式的"分散10只各10%"**。

```
  ┌─────────────────┬──────────┬──────────────────┐
```

```python
def get_position_tier(capital: float) -> dict:
    """根据本金自动返回仓位参数"""
    if capital <= 20000:
        return {
            'tier': '超小资金',
            'max_stocks': 2,
            'max_single_pct': 0.50,
            'stop_loss_pct': 0.03,
            'max_price': 30,          # 股价上限
            'prefer_etf': True,
            'total_position': {'strong': 1.0, 'shaky': 0.5, 'weak': 0.0}
        }
    elif capital <= 50000:
        return {
            'tier': '小资金',
            'max_stocks': 4,
            'max_single_pct': 0.30,
            'stop_loss_pct': 0.05,
            'max_price': 100,
            'total_position': {'strong': 0.9, 'shaky': 0.4, 'weak': 0.2}
        }
    elif capital <= 100000:
        return { ... }  # 5-10万档
    else:
        return { ... }  # 10万+标准档

def calculate_position(signal, capital, tier, latest_price):
    """计算实际可买入的股数"""
    # 1. 按百分比计算目标金额
    target_amount = capital * tier['max_single_pct']
    
    # 2. 转成手数（向下取整）
    lots = int(target_amount / (latest_price * 100))
    
    if lots == 0:
        return {
            'actionable': False,
            'reason': f'资金不足：1手需{latest_price*100:.0f}元，'
                      f'单票上限{target_amount:.0f}元'
        }
    
    # 3. 实际买入金额
    actual_amount = lots * 100 * latest_price
    actual_pct = actual_amount / capital
    
    # 4. 如果实际占比超上限，给出警告但允许买入（小资金妥协）
    warning = None
    if actual_pct > tier['max_single_pct']:
        warning = f'⚠️ 实际占比{actual_pct:.0%}，超过上限{tier["max_single_pct"]:.0%}，'
        warning += '小资金无法精确控制，请确认接受此风险'
    
    return {
        'actionable': True,
        'lots': lots,
        'shares': lots * 100,
        'amount': actual_amount,
        'pct': actual_pct,
        'stop_loss_price': round(latest_price * (1 - tier['stop_loss_pct']), 2),
        'warning': warning
    }
```

### 5.4 止损规则（持仓阶段）

信号发出、买入之后，止损就开始工作了。这是防守的最后一道保险。

#### 三种止损方式

```
┌──────────────┬──────────────────┬─────────────────────┐
│ 止损类型      │ 触发条件          │ 优缺点               │
├──────────────┼──────────────────┼─────────────────────┤
│ 固定比例止损  │ 亏损达到 -5%      │ ✅简单粗暴，最容易执行 │
│              │                  │ ❌可能被洗出去        │
├──────────────┼──────────────────┼─────────────────────┤
│ 均线止损      │ 收盘价跌破 MA60   │ ✅趋势彻底走坏才出    │
│              │                  │ ❌幅度可能超过 -10%   │
├──────────────┼──────────────────┼─────────────────────┤
│ 时间止损      │ 买入后10个交易日   │ ✅释放被套资金        │
│              │ 仍没有盈利        │ ❌可能错过即将的突破   │
└──────────────┴──────────────────┴─────────────────────┘
```

#### 推荐组合：新手用"固定比例 + 时间"双止损

```

1. 固定止损（硬止损）：
```

### 5.5 风控如何融入每日流程

```
每日 15:05 脚本运行时的风控检查顺序：

1. 拉取大盘指数数据
2. 计算大盘状态 → 得到仓位系数
3. 遍历策略，生成原始信号
4. 防线一：大盘状态过滤
   - 强势/震荡 → 信号继续
   - 弱势/极弱 → 买入信号全部屏蔽
5. 防线二：基础规则过滤
   - ST/涨停/停牌 → 排除
6. 防线三：仓位计算
   - 单票≤15%
   - 总仓位≤当前市场仓位上限
7. 检查现有持仓的止损状态
   - 哪些触发止损了？→ 加入卖出信号
8. 汇总 → 格式化 → 推送手机
```

```
最终目标：

  ⚠️ 资金增长后记得更新 TOTAL_CAPITAL，
```

## 9. 策略模块设计（可插拔）

### 9.1 内置策略清单

系统自带3个简单策略作为起点。规律：**简单的策略往往比复杂的更稳健**。

#### 策略1：双均线趋势跟踪

```
逻辑：
  MA(10) 当日线上穿 MA(30) 当日线 → BUY
  MA(10) 当日线下穿 MA(30) 当日线 → SELL
  否则 → HOLD

参数（可配置）：
  fast_period: 10    # 快线周期
  slow_period: 30    # 慢线周期

筛选条件：
  只在沪深300成分股中筛选（流动性好、基本面ok）

历史表现（供参考，非承诺）：
  沪深300上，2019-2024回测年化约8-12%
  震荡市（如2022）表现差，趋势市（2020）表现好
```

#### 策略2：动量突破

```
逻辑：
  今日收盘价 > 过去20日最高收盘价 → BUY（突破买入）
  今日收盘价 < 过去20日最低收盘价 → SELL（破位卖出）
  否则 → HOLD

参数（可配置）：
  lookback: 20       # 回顾天数
  buffer: 0.02       # 缓冲区（突破2%才触发，减少假突破）

关键点：
  - 缓冲区很重要：不加缓冲在震荡市会频繁假突破
  - 趋势市中表现好，震荡市中需要buffer保护
```

#### 策略3：均值回归（布林带）

```
逻辑：
  收盘价 < 布林带下轨(20,2) → BUY（超卖回归）
  收盘价 > 布林带上轨(20,2) → SELL（超买回归）
  收盘价回到中轨(20日均线) → 平仓

参数（可配置）：
  period: 20          # 中轨周期
  std_dev: 2.0        # 标准差倍数

风险提示：
  - 单边趋势中会反复抄底被套
  - 需要结合趋势过滤（如：只在MA60向上时才做多）
```

### 9.2 新手策略选择建议

```
建议从策略1（双均线）开始，原因：
✅ 逻辑最简单，容易理解和信任
✅ 参数少（只有2个），不易过拟合
✅ 信号明确（交叉那一刻），不会模棱两可
✅ 是趋势策略的基石，理解了它，其他趋势策略都能理解
✅ 在趋势市表现稳健，2024-2025年A股有一定趋势特征

⚠️ 但要清楚它的缺点：
  - 震荡市频繁打脸（连续小亏）
  - 滞后于价格（均线天然滞后）
  - 不可能抓到顶部和底部
```

```sql
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    date        DATE,                -- 信号生成日期
    code        TEXT,
    name        TEXT,
    strategy    TEXT,                -- 策略名称
    action      TEXT,                -- BUY / SELL
    strength    REAL,                -- 信号强度 0-1
    reason      TEXT,                -- 信号原因
    price       REAL,                -- 信号触发价格
    executed    INTEGER DEFAULT 0    -- 是否跟单
);

-- 成交记录表（手动录入）
CREATE TABLE trade_log (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    buy_date    DATE,
    sell_date   DATE,                -- NULL表示尚未卖出
    code        TEXT,
    name        TEXT,
    buy_price   REAL,
    sell_price  REAL,
    quantity    INTEGER,             -- 股数
    amount      REAL,                -- 成交金额
    commission  REAL,                -- 手续费
    pnl         REAL,                -- 盈亏金额（卖出后填入）
    pnl_pct     REAL,                -- 盈亏百分比
    strategy    TEXT,                -- 跟的哪个策略
    note        TEXT                 -- 备注
);

-- 持仓表（自动计算）
CREATE VIEW current_position AS
SELECT 
    code, name, buy_date, buy_price, quantity,
    (SELECT close FROM daily_kline WHERE code = t.code 
     ORDER BY date DESC LIMIT 1) as latest_price,
    (SELECT close FROM daily_kline WHERE code = t.code 
     ORDER BY date DESC LIMIT 1) * quantity - buy_price * quantity as unrealized_pnl
FROM trade_log t
WHERE sell_date IS NULL;
```

### 10.2 数据更新策略

```
每日增量更新流程：

1. 查询本地数据库最新日期
   SELECT MAX(date) FROM daily_kline

2. 对比今天日期
   如果本地最新 = 今天 → 今日数据已更新，跳过
   如果本地最新 = 昨天 → 只下载明天（今天）的数据
   如果本地最新 < 昨天 → 补充下载缺失日期

3. AKShare下载增量数据
   只下载 from_date → to_date 的数据

4. 写入SQLite
   使用 INSERT OR REPLACE 避免重复

5. 日志记录
   2026-06-06 15:05:03 Downloaded 2436 rows of daily data
   2026-06-06 15:05:04 Inserted 2436 rows into daily_kline
```

### 11.3 风险提示清单

> 以下风险的应对方案详见 [第5节：风控体系](#5-风控体系三道防线)。

```
策略风险：
□ 策略失效风险 —— 过去有效不代表未来有效
□ 过拟合风险 —— 策略偷看过历史数据
□ 风格切换风险 —— 市场风格变了，策略不适应
  → 防线一（大盘择时）能部分缓解风格切换的影响

执行风险：
□ 信号滞后风险 —— 收盘生成信号，第二天开盘可能高开/低开
□ 流动性风险 —— 小盘股可能买不到或卖不掉
  → 防线二（基础过滤）排除流动性差的标的
□ 涨停/跌停风险 —— 涨停买不到，跌停卖不掉
  → 防线二（基础过滤）自动排除
□ T+1隔夜风险 —— 今天买入，碰到利空只能明天卖
  → 防线三（单票15%上限）控制单次隔夜风险

技术风险：
□ 数据源中断 —— AKShare偶尔不稳定
□ 脚本运行失败 —— 某天忘记跑了
□ 通知未送达 —— 手机没电/网络问题

人性风险（最大风险）：
□ 选择性跟单 —— 只跟盈利信号，不跟亏损信号
  → 解决：记录每笔信号，每周对比"全跟"vs"选择性跟"的收益
□ 中途干预 —— 盘中手动改止损/止盈
  → 解决：买入时就设好止损价，不要盘中临时改
□ 情绪波动 —— 连亏5笔后不敢跟，或狂跟想翻本
  → 解决：仓位控制限制单笔亏损，5%止损×15%仓位=总资金仅亏0.75%
□ 不记录 —— 不记盈亏，不知道自己到底赚了还是亏了
  → 解决：盈亏跟踪模块强制记录，不记录就不知道策略好不好
```
