## 1. 什么是量化交易

### 一句话定义

**量化交易**是用数学模型和计算机程序来替代（或辅助）人工做出交易决策的方式。

### 通俗理解

传统交易者靠"盘感"、经验、消息来买卖。量化交易者靠数据、统计、代码来买卖。

打个比方：
- 传统交易像老中医——望闻问切，凭经验开方
- 量化交易像西医检验——抽血化验，指标超过阈值就用药

### 核心要素

```
量化交易 = 数据 + 模型 + 纪律 + 概率优势
```

| 要素 | 说明 |
|------|------|
| **数据** | 股价、成交量、财务数据、宏观经济指标……一切可以量化的信息 |
| **模型** | 用数学/统计/机器学习方法从数据中提取交易信号 |
| **纪律** | 机器严格执行规则，不受恐惧、贪婪、犹豫的影响 |
| **概率优势** | 不追求每笔都赢，而是追求大量交易后，赢的比输的多 |

### 量化交易的三种形态

```
┌──────────────────────────────────────────────────────────────┐
│  手工交易         半自动化           全自动量化              │
│  ─────────       ─────────          ─────────               │
│  人看盘 →       量化筛选 →          策略自动生成信号 →       │
│  人决策 →       人确认 →            程序自动下单 →           │
│  人下单          半自动下单          程序自动风控             │
│                                                              │
│  完全主观        人机协作            完全程序化               │
└──────────────────────────────────────────────────────────────┘
```

我们最终要做的，就是最右边那种——全自动量化交易系统。

## 2. 量化的核心优势与局限

### 优势

| 优势 | 说明 |
|------|------|
| **纪律性** | 不会因为亏了钱就"上头"追涨杀跌，也不会因为赚了钱就得意忘形 |
| **广度** | 人同时只能盯几只股票，程序可以同时监控全市场5000+只股票 |
| **速度** | 从信号出现到下单，人可以快到秒级，程序可以快到微秒级 |
| **可验证** | 任何想法都可以用历史数据回测，用数据说话，而不是"我觉得" |
| **可复制** | 一套策略可以在多账户、多品种上同时运行 |
| **概率思维** | 量化天然用大数定律思考，不纠结单笔盈亏 |

### 局限（必须正视）

| 局限 | 说明 |
|------|------|
| **过拟合** | 策略在历史数据上表现完美，实盘一塌糊涂——把噪声当成了规律 |
| **黑天鹅** | 模型基于历史，但历史不会简单重复，极端事件往往超出模型预期 |
| **容量限制** | 好策略的资金容量有限，钱太多反而跑不动 |
| **策略衰退** | 市场会进化，曾经赚钱的策略会慢慢失效（Alpha Decay） |
| **技术风险** | 程序Bug、网络中断、数据错误都可能导致灾难性损失 |
| **A股特殊限制** | T+1、涨跌停、融券困难、做空手段有限 |

## 3. 量化交易的完整工作流

```
                    量化交易生命周期

  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │          │    │          │    │          │
  │  1.数据   │───▶│ 2.因子/  │───▶│ 3.策略   │
  │  获取     │    │   信号   │    │   构建   │
  │          │    │          │    │          │
  └──────────┘    └──────────┘    └──────────┘
                                       │
                                       ▼
  ┌──────────┐    ┌──────────┐    ┌──────────┐
  │          │    │          │    │          │
  │  6.实盘   │◀───│ 5.仿真   │◀───│ 4.回测   │
  │   运行    │    │   交易   │    │   验证   │
  │          │    │          │    │          │
  └──────────┘    └──────────┘    └──────────┘
       │
       ▼
  ┌──────────┐    ┌──────────┐
  │          │    │          │
  │  7.监控   │───▶│ 8.迭代   │──▶ 回到步骤2
  │   评估    │    │   优化   │
  │          │    │          │
  └──────────┘    └──────────┘
```

### 各环节详解

#### 第一步：数据获取
- 从哪里拿数据？（数据源）
- 数据干不干净？（清洗、复权、对齐）
- 数据怎么存？（数据库设计）
- 数据怎么取？（API设计）

#### 第二步：因子/信号挖掘
- **因子**：对某只股票未来收益有预测能力的量化指标
- 例如：市盈率（PE）、动量（过去N天涨幅）、换手率……
- 单因子 → 检验有效性 → 多因子组合 → 生成交易信号

#### 第三步：策略构建
- 什么时候买？什么时候卖？买多少？卖多少？
- 把多个信号的逻辑组合成完整的交易规则

#### 第四步：回测验证
- 用历史数据模拟实盘交易
- 关注：收益率、最大回撤、夏普比率、胜率、盈亏比……
- 警惕：过拟合、未来函数、幸存者偏差

#### 第五步：仿真交易（纸交易）
- 用实时行情模拟交易，但不涉及真金白银
- 验证策略在"当下"是否仍然有效

#### 第六步：实盘运行
- 接通券商API，真金白银自动交易
- 需要：极低的故障率、完善的监控告警

#### 第七步：监控评估
- 每日检查：账户状态、持仓、盈亏
- 定期评估：策略是否偏离预期？是否需要干预？

#### 第八步：迭代优化
- 根据实盘表现和新数据，持续改进策略
- 这是一个永无止境的循环

## 4. 数据：量化的血液

> 垃圾进，垃圾出。数据质量决定了策略的天花板。

### 4.1 数据类型

```
行情数据
├── 日线数据（开、高、低、收、量、额）
├── 分钟线数据（1min, 5min, 15min, 30min, 60min）
├── Tick数据（每笔成交明细，最深度的数据）
└── Level-2数据（十档买卖盘口、逐笔成交）

基本面数据
├── 财务报表（利润表、资产负债表、现金流量表）
├── 估值指标（PE、PB、PS、PCF……）
├── 盈利预测（分析师一致预期）
└── 股本结构（流通股、限售股、大股东持股）

另类数据（Alternative Data）
├── 新闻舆情（NLP情感分析）
├── 龙虎榜数据
├── 融资融券数据
├── 北向资金（沪深港通）
├── 大宗交易
└── 社交媒体情绪
```

### 4.3 数据存储建议

```
原始数据层（Raw）
├── 按数据源、日期、品种原样存储
├── 格式：Parquet（压缩比高、读取快）
└── 永远保留原始数据，便于回溯

清洗数据层（Clean）
├── 复权处理（前复权/后复权）
├── 停牌处理、缺失值填充
├── 异常值检测与修正
└── 多源数据对齐

特征数据层（Features）
├── 计算好的因子值
├── 按日期-股票索引
└── 供策略直接调用
```

## 5. 策略：量化的灵魂

### 5.1 策略分类全景

```
策略分类

├── 趋势跟踪（Trend Following）
│   ├── 均线交叉（金叉死叉）
│   ├── 通道突破（唐奇安通道、布林带）
│   └── 动量策略（追涨强势股）
│
├── 均值回归（Mean Reversion）
│   ├── 统计套利（配对交易）
│   ├── 布林带回归
│   └── RSI超买超卖
│
├── 因子选股（Factor Investing）
│   ├── 价值因子（低PE、低PB）
│   ├── 动量因子（短期/中期动量）
│   ├── 质量因子（高ROE、低杠杆）
│   ├── 规模因子（小盘股效应）
│   ├── 波动率因子（低波动异象）
│   └── 多因子组合
│
├── 事件驱动（Event Driven）
│   ├── 业绩公告（超预期买入）
│   ├── 分红送转（填权行情）
│   ├── 重组/定增
│   └── 指数调仓（纳入/剔除）
│
├── 高频交易（HFT，A股受限）
│   ├── 做市策略
│   ├── 套利策略（跨期、跨品种）
│   └── 订单流预测
│
├── 机器学习策略
│   ├── 树模型（XGBoost, LightGBM）
│   ├── 神经网络（LSTM预测股价）
│   ├── 强化学习（训练交易智能体）
│   └── NLP（新闻情绪选股）
│
└── 资产配置/组合管理
    ├── 风险平价
    ├── 均值-方差优化
    └── 行业轮动
```

### 5.2 几个经典策略速览

#### 策略一：双均线趋势跟踪

```
最简单的趋势策略：
  当 短期均线（如MA20）上穿 长期均线（如MA60）→ 买入
  当 短期均线（如MA20）下穿 长期均线（如MA60）→ 卖出

优点：简单、直观、不容易过拟合
缺点：震荡市频繁止损、滞后于价格
```

#### 策略二：多因子选股

```
每月末：
  1. 计算所有股票的：PE倒数（价值）、过去12个月涨幅（动量）、ROE（质量）
  2. 每只股票按每个因子排名，加权得综合得分
  3. 买入综合得分最高的30只股票，等权重配置
  4. 每月调仓一次

优点：逻辑扎实、容量大、可解释性强
缺点：因子可能失效（拥挤交易）、调仓成本
```

#### 策略三：配对交易（统计套利）

```
找到历史上高度相关的两只股票（如同行业的招行和兴业）：
  当它们的价差偏离均值超过2个标准差 → 做多弱势股，做空强势股
  当价差回归均值 → 平仓获利

A股问题：做空困难（融券券源少）
变通方案：只做多价差回归，不做空（收益减半但可行）
```

### 5.3 策略设计原则

1. **逻辑先行**：策略必须有合理的经济学/行为金融学解释，不能纯靠数据挖掘
2. **简单优于复杂**：能用3个参数就不要用30个，参数越少越不容易过拟合
3. **先求不亏，再求盈利**：防御性设计（风控）比攻击性设计（收益）更重要
4. **考虑现实约束**：手续费、滑点、涨跌停、T+1、流动性……这些必须纳入模型

## 6. 回测：量化的试金石

### 6.1 回测的本质

回测就是用历史数据回答一个假设性问题：
> "如果我在2015年1月1日开始用这套规则交易，到现在会赚多少钱？"

### 6.2 回测的核心指标

| 指标 | 公式/含义 | 说明 |
|------|-----------|------|
| **累计收益率** | (最终净值-1)×100% | 总赚了多少 |
| **年化收益率** | 累计收益年化 | 便于不同周期比较 |
| **最大回撤(MDD)** | 从最高点到最低点的最大跌幅 | 你能承受的最大痛苦 |
| **夏普比率** | (年化收益-无风险利率)/年化波动率 | 每单位风险换来的超额收益，>1算合格，>2算优秀 |
| **卡玛比率** | 年化收益/最大回撤 | 性价比指标，>1算不错 |
| **胜率** | 盈利交易次数/总交易次数 | 不要纠结这个，30%胜率也可以很赚钱 |
| **盈亏比** | 平均盈利/平均亏损 | 胜率低但盈亏比高，照样赚钱 |
| **最大连续亏损次数** | 最差情况连续亏了多少笔 | 心理承受力的底线 |

### 6.3 回测中最常见的陷阱

> ⚠️ 这可能是量化中最重要的一节——避免自欺欺人。

#### 陷阱1：未来函数（Look-ahead Bias）

```
错误示例：用"今天发布的财报"来预测"财报发布前的股价"
正确做法：财报数据只能用在发布日期之后，不能"穿越"

检查方法：你的策略中使用的每一个数据，在那个时间点是否真实可得？
```

#### 陷阱2：幸存者偏差（Survivorship Bias）

```
错误示例：回测时只用了"现在还存活"的股票
正确做法：回测时每个时间点都要包含当时上市的所有股票，包括后来退市的

后果：高估收益20%-50%不等
```

#### 陷阱3：过拟合（Overfitting）

```
灵魂问答：
Q: "你这策略夏普3.5，年化50%，最大回撤5%，怎么做到的？"
A: "我加了50个参数，在100只股票上反复试了1000次"
Q: "……这策略实盘能用吗？"
A: "不能。"


过拟合 = 把历史的噪声（随机波动）当成规律来拟合
结果 = 回测曲线完美，实盘一塌糊涂

防范方法：
- 减少参数数量
- 样本外测试（拿一部分数据完全不看，最后验证）
- 交叉验证（不同时间段）
- 逻辑约束（参数必须在合理范围内）
```

#### 陷阱4：忽略了交易成本

```
A股实际交易成本：
├── 印花税：卖出时 0.05%（2023年8月起减半征收）
├── 佣金：约 0.01%-0.03%（双向）
├── 过户费：0.001%（双向）
├── 滑点：实际成交价与预期价的差距
└── 冲击成本：大资金买卖时推高/压低价格

高频策略尤其敏感，忽略成本会让回测收益虚高到离谱
```

#### 陷阱5：前视偏差的各种变体

```
- 用全样本做标准化 → 错误，应该用滚动窗口
- 用全样本分位数排名 → 错误，应该按时间点排名
- 停牌日当成正常交易日 → 错误，停牌日无法交易
- 涨停价买入 → 错误，涨停买不到
- 跌停价卖出 → 错误，跌停卖不掉
```

## 7. 风控：量化的安全带

> 策略决定你能赚多少，风控决定你能活多久。

### 7.1 风控的三个层次

```
第一层：策略层面
├── 止损：（单笔亏损达到X%强行卖出）
├── 止盈：（单笔盈利达到Y%锁定利润）
├── 持仓时间限制：（持有N天后无论盈亏都平仓）
└── 仓位上限：（单只股票不超过总资金的Z%）

第二层：组合层面
├── 行业分散：（单一行业不超过总资金的A%）
├── 市值分散：（大/中/小盘均衡）
├── 最大同时持仓数：（比如最多持有20只股票）
└── 整体仓位管理：（牛市中高仓位，熊市中低仓位）

第三层：系统层面
├── 日内最大亏损限制：（今天亏了B%就关机，明天再说）
├── 连续回撤熔断：（连续回撤超过C%暂停策略）
├── 资金曲线监控：（资金曲线跌破N日均线 → 减仓）
└── 人工干预开关：（紧急情况一键清仓）
```

### 7.2 A股特殊风控要点

| 约束 | 影响 |
|------|------|
| **T+1** | 今天买的明天才能卖，策略必须考虑隔夜风险 |
| **涨跌停板** | 主板±10%，创业板/科创板±20%，可能买不到或卖不掉 |
| **做空困难** | 融券券源非常有限，大部分时候只能做多赚钱 |
| **停牌** | 可能突然停牌数周甚至数月，资金被锁 |
| **印花税/佣金** | 高频换手会被交易成本吃掉利润 |

## 8. 实盘：量化的终点与起点

### 8.1 从回测到实盘的鸿沟

```
回测世界                vs        实盘世界
──────────────────────────────────────────────
知道未来数据           vs        只能看到过去
完美成交               vs        可能买不到/卖不掉
没有情绪               vs        亏了真的会心疼
零成本                 vs        手续费、滑点
历史重演               vs        未来充满不确定性
```

## 11. 常见误区与忠告

### 11.1 新手最容易犯的错

| 误区 | 现实 |
|------|------|
| **"回测年化100%，我要发财了！"** | 几乎肯定有过拟合或未来函数，回头仔细检查 |
| **"越复杂的模型越好"** | 越复杂越容易过拟合，简单策略往往更稳健 |
| **"亏了是因为参数没调好"** | 可能是策略逻辑本身有问题，不是参数的事 |
| **"回测好了直接满仓干"** | 先仿真交易、先小资金、先跑几个月再说 |
| **"量化就是躺着赚钱"** | 真正的量化需要持续的研究、维护和迭代 |
| **"找到圣杯了，不用再研究了"** | 没有永远赚钱的策略，市场永远在进化 |
| **"机器学习可以自己找到赚钱规律"** | ML是工具不是魔法，垃圾进垃圾出 |

### 11.2 实在的忠告

#### 关于收益预期

```
合理预期：
- 年化10%-15%：已经不错了（跑赢大多数理财产品）
- 年化15%-25%：相当优秀（顶级量化基金的水平）
- 年化25%-50%：要么是天才，要么是还没遇到黑天鹅
- 年化50%+：99.9%是过拟合/造假/短期运气

如果一个策略告诉你年化50%回撤5%，请保持高度怀疑。
```

#### 关于资金管理

```
不要：
- 借钱炒股
- 把生活费投进去
- all in 一个策略
- 一上来就大资金

要：
- 只用闲钱（亏光了也不影响生活）
- 分散到多个不相关的策略
- 从小资金开始，逐步增加
- 永远留有余地
```

#### 关于心态

```
量化交易的本质是用纪律对抗人性，但讽刺的是——
你需要极大的人性纪律来坚持量化交易。

当策略连续亏损10次时，你还能坚持执行吗？
当身边人都在追涨大赚时，你能坚持你的因子选股吗？
当市场风格切换，你的策略失效半年时，你会放弃吗？

量化交易最大的敌人不是市场，是你自己。
```
