### Phase 1：多因子选股（v0.3.0 核心）

**为什么是多因子**：
- 是专业量化最主流的方法，有海量学术和业界验证
- 月频调仓，不需要盯盘
- 和我们现有架构天然匹配（SQLite + 回测引擎 + 钉钉推送）
- 不依赖做空，纯多头 A 股适用

**具体内容**：

```
1. 数据扩展
   ├── 财务数据：PE、PB、ROE、营收增速、利润增速
   ├── 数据源：AKShare stock_financial_abstract()
   └── 新增表：financial_data(code, date, pe, pb, roe, revenue_growth, profit_growth)

2. 因子计算（每月）
   ├── 价值因子：PE倒数（1/PE）排名
   ├── 动量因子：过去12个月涨幅（剔除最近1个月）
   ├── 质量因子：ROE排名
   └── 可选：低波动因子（近60日波动率）

3. 因子合成
   ├── 标准化：每个因子转成 0-100 分
   ├── 等权合成：三个因子简单平均
   └── 后续可升级：IC加权、机器学习加权

4. 选股调仓
   ├── 每月初计算因子得分
   ├── 按资金档位决定持仓数（见下方资金方案）
   └── 每月调仓一次

5. 回测验证
   ├── 训练集：2019-2022（研究因子、调参）
   ├── 验证集：2023-2024（验证效果）
   └── 测试集：2025-2026（最终评估，只用一次）
```

### 频率决策：月频 ✅

```
为什么选月频：
  1. 财务数据本身是季度/月度更新的，日频没有额外信息
  2. A股月度动量和反转效应最稳定（券商研报验证）
  3. 月频交易成本低（12次/年 vs 日频 200+次/年），手续费吃掉更少
  4. 回测 84 个月样本足够统计显著
  5. 不需要盯盘——每月看一眼即可
```

### 资金分档方案

┌────────────┬──────────┬──────────────────────────────┐

## 三、标准开发流程

### 3.1 策略开发流程

以后任何新策略都按这个流程：

```
1. 研究阶段
   □ 读资料（论文/研报/GitHub）
   □ 理解逻辑（为什么能赚钱？在什么市场有效？）
   □ 评估可行性（数据能不能拿到？资金够不够？）

2. 数据准备
   □ 确认数据源（AKShare / 聚宽 / Tushare）
   □ 写下载脚本 → SQLite
   □ 验证数据质量（缺失率、异常值）

3. 因子/信号开发
   □ 实现因子计算
   □ 单因子 IC 分析（信息系数，衡量因子预测力）
   □ 分层回测（Top/Bottom 分组看收益差异）

4. 回测验证
   □ 训练集上开发和调参（2019-2022）
   □ 验证集上评估（2023-2024）
   □ 测试集最终确认（2025-2026，只用一次）
   □ 蒙特卡洛验证统计可靠性

5. 上线部署
   □ 集成到 run.py 或独立脚本
   □ 加钉钉推送格式
   □ 本地测试 3 天
   □ 服务器 git pull + 手动跑一次验证
```

### 3.2 参数调优流程

```
1. 在训练集上调参（改参数、看反馈）
2. 在验证集上评估（不行就回去步骤1）
3. 验证集满意 → 测试集跑一次（只看不调）
4. 三个区间表现一致 → 参数可靠
5. 三个区间表现差异大 → 过拟合，放弃这组参数

禁止：
  ❌ 对着测试集调参（数据泄露）
  ❌ 三个区间加一起调参（自欺欺人）
  ❌ 回测好得离谱就不做蒙特卡洛（可能是运气）
```
