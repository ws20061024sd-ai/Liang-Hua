## 一、回测方法论

### 1.1 为什么本地回测而不是聚宽？

```
本地回测的优势：
  ✅ 和实盘策略代码完全一致（同一套 MaCrossStrategy 类）
  ✅ 数据源一致（新浪财经 → SQLite，和每天 cron 拉的是同一个源）
  ✅ 可自定义任何细节（手费率、止损逻辑、择时规则）
  ✅ 结果可复现（代码和数据都在仓库里）

聚宽回测的局限性：
  ❌ 策略代码需要重新适配聚宽API，不能直接复用
  ❌ 数据源不同，结果和实盘可能有偏差
  ❌ 无法自定义交易细节
```

### 1.2 回测参数

| 参数 | 值 | 说明 |
|------|----|------|
| **策略** | 双均线趋势跟踪 | strategies/ma_cross.py |
| **快线周期** | 10 日 | MA10 |
| **慢线周期** | 30 日 | MA30 |
| **股票池** | 沪深300成分股 | 300只 |
| **回测区间** | 2023-06-06 ~ 2026-06-08 | 3年，727个交易日 |
| **初始资金** | ¥100,000 | 10万元 |
| **手续费** | 0.03% 买卖 + 0.05% 印花税卖出 | 合计约 0.08%/笔 |
| **最大持仓** | 10 只 | 单只 10% 仓位 |
| **止损** | -5%（默认） / -3%（紧密） | 收盘价跌破成本价时触发 |
| **大盘择时** | 指数 < MA20 时不买 | engine/market_timing.py |

### 1.3 回测局限

| 局限 | 影响 |
|------|------|
| 幸存者偏差 | 用了当前沪深300成分股回看过去，历史退市/调出的股票不在池中 |
| 3年数据 | 未覆盖完整牛熊周期（未包含 2019-2020 牛市） |
| 日线级别 | T+1 无法在当天交易，未模拟次日开盘滑点 |
| 停牌处理 | 简化处理，停牌日跳过 |

## 二、回测结果

### 2.1 方案一：纯双均线（无大盘择时）

```
回测区间：2023-06-06 ~ 2026-06-08（3年）
```

| 指标 | 数值 | 评价 |
|------|:----:|------|
| 累计收益率 | **-26.6%** | ❌ 亏损严重 |
| 年化收益率 | **-10.2%** | ❌ 大幅跑输基准 |
| 最大回撤 | **46.2%** | ❌ 接近腰斩 |
| 夏普比率 | **-0.12** | ❌ 承担风险没换到收益 |
| 胜率 | **35.9%** | ⚠️ 正常（趋势策略 30-40% 常见） |
| 交易次数 | **298** | 平均每2.4天一笔 |
| 平均盈利 | **+12.7%** | ✅ 赚一次赚得大 |
| 平均亏损 | **-4.9%** | ⚠️ 止损有效 |
| 盈亏比 | **2.59** | ✅ 赚一次够亏 2.5 次 |

**解读**：

```
策略核心逻辑是对的——盈亏比 2.59，趋势捕捉能力不错。
问题出在哪里？没有区分市场环境。

在 2023-2024 的下跌市中，策略反复发金叉信号
→ 买入 → 跌 → 止损 → 再发金叉 → 再亏
→ 81次本该跳过的亏损交易 → 累积亏损吞噬了牛市的利润
```

```
回测区间：同方案一
择时规则：沪深300 < MA20 时不买入
```

| 指标 | 数值 | 对比方案一 |
|------|:----:|:----------:|
| 累计收益率 | **-3.0%** | +23.6% |
| 年化收益率 | **-1.1%** | +9.1% |
| 最大回撤 | **44.1%** | -2.1% |
| 夏普比率 | **0.16** | +0.28 |
| 胜率 | **35.9%** | 不变 |
| 交易次数 | **217** | **少亏81次** |
| 盈亏比 | **6.86** | 翻 2.6 倍 |

**解读**：

```
择时省了 81 次交易，几乎全是亏损交易。
盈亏比从 2.59 → 6.86，择时过滤掉的恰恰是那些"进去就亏"的假信号。

但最大回撤改善不明显（46%→44%），因为：
  - 回测期间 A 股整体下跌，持仓中的亏损仍在累积
  - 3 年数据中有 2 年处于下行周期
  - 需要更长的回测区间（5-7年）才能看到择时对回撤的真正改善
```

## 三、各指标详解

### 3.1 累计收益率

```
公式：(最终净值 - 1) × 100%

表示：从回测开始到结束，总共赚了/亏了多少钱

方案一：10万 → 7.34万（亏了26.6%）
方案二：10万 → 9.70万（亏了3.0%）
```

**标准**：
| 范围 | 评价 |
|------|------|
| >0% | 赚钱 |
| >20%/年 | 优秀 |
| <0% | 策略需要改进或放弃 |

### 3.2 年化收益率

```
公式：(最终净值)^(1/年数) - 1

表示：折算到每年的收益率，便于不同回测周期比较

方案一：-10.2%/年
方案二：-1.1%/年
```

**标准**：
| 范围 | 评价 |
|------|------|
| >15% | 优秀 |
| 10-15% | 良好 |
| 5-10% | 及格 |
| <5% | 不如理财 |
| <0% | 亏钱策略 |

### 3.3 最大回撤

```
定义：从资金曲线最高点到随后最低点的最大跌幅

方案一：46.2% → 10万最多时候只剩 5.38万
方案二：44.1% →  10万最多时候只剩 5.59万
```

**这是最重要的指标**——你心理上能承受的最大的亏损幅度。

**标准**：
| 范围 | 评价 |
|------|------|
| <10% | 优秀 |
| 10-20% | 良好 |
| 20-30% | 可接受 |
| 30-50% | 需要改进 |
| >50% | 不可接受 |

### 3.4 夏普比率

```
公式：(年化收益 - 无风险利率) / 年化波动率

表示：每承担 1 单位风险，能换回来多少超额收益

方案一：-0.12 → 承担了风险，没收益
方案二： 0.16 → 稍微正收益
```

**标准**：
| 范围 | 评价 |
|------|------|
| >2.0 | 优秀（顶级量化基金水平） |
| 1.0-2.0 | 良好 |
| 0.5-1.0 | 及格 |
| <0.5 | 表现不佳 |
| <0 | 不如存银行 |

### 3.5 胜率

```
公式：盈利交易次数 / 总交易次数 × 100%

方案一：35.9%（298笔，107笔盈利）
方案二：35.9%（217笔，78笔盈利）
```

> ⚠️ 新手最常误解的指标。**胜率低不代表不赚钱**——趋势策略天然低胜率（30-40%），因为大部分信号都是假突破。但只要盈亏比高（赚一次够亏好几次），总体就是赚的。

**标准**：趋势策略 30-40% 正常；均值回归 50-60% 正常。

### 3.6 盈亏比

```
公式：平均盈利 / |平均亏损|

表示：每亏 1 块钱，你能赚几块钱回来

方案一：2.59 → 赚一次够亏 2.59 次
方案二：6.86 → 赚一次够亏 6.86 次
```

> 这是最重要的收益质量指标。盈亏比 > 2 才值得做。

**标准**：
| 范围 | 评价 |
|------|------|
| >3 | 优秀 |
| 2-3 | 良好 |
| 1.5-2 | 及格 |
| <1.5 | 不值得做 |

## 四、关键结论

### 4.1 策略有效性的证据

```
✅ 盈亏比 2.59~6.86：赢一次够亏多次，逻辑是对的
✅ 大盘择时省了 81 次亏损交易：择时规则有效
✅ 胜率 35.9% 稳定：趋势策略的典型表现
```

### 4.2 策略的问题

```
❌ 整体还是亏钱的：回测期间 A 股大环境不好
❌ 最大回撤太大：44-46% 不能被接受
❌ 缺少更多牛市数据：无法判断在好行情中能赚多少
```

### 4.3 改进方向

| 优先级 | 改进 | 对哪个指标有利 |
|:--:|------|:--:|
| 1 | 扩展回测区间到 2019-2026 | 拿到完整牛熊数据 |
| 2 | 加入成交量过滤 | 减少假金叉 |
| 3 | 均线斜率过滤（MA20 必须向上） | 提高胜率 |
| 4 | 加入行业/风格分散 | 降低回撤 |
| 5 | 多策略并行（动量突破+均值回归） | 提高稳定性 |

## 五、与聚宽回测的对比

| 维度 | 本地回测（本报告） | 聚宽回测 |
|------|:--:|:--:|
| 策略代码 | 和实盘完全一致 | 需要改写 |
| 数据源 | 新浪（同实盘） | 聚宽自有数据 |
| 回测区间 | 3年（受限本地数据量） | 可达10年+ |
| 手续费模拟 | 0.08%/笔 | 0.1%/笔 |
| 自定义 | 完全自由 | 受限平台框架 |
| 适用场景 | 验证代码逻辑 | 验证策略思想 |

**建议**：两者互补——本地回测验证代码正确性，聚宽回测验证策略长期表现。
