## 一、项目概述

### 1.1 这是什么

一套**半自动化 A 股量化交易系统**。系统每天自动分析沪深 300 成分股，生成买卖信号和市场日报，推送到用户手机钉钉。用户查看信号后，**手动在中信建投 APP 下单**，系统不自动交易。

### 1.2 核心设计

```
数据下载 → 策略计算 → 三道防线过滤 → 信号汇总 → 钉钉推送
                                                ↓
                                         用户手机查看
                                                ↓
                                    人工判断 → 券商APP手动下单
```

### 1.3 当前状态

- 3 个策略并行运行，部署在腾讯云服务器
- 工作日 21:00 自动推送交易信号，21:05 推送市场日报
- 用户资金约 1 万元（小资金档位）
- 尚未开始实盘跟单，处于观察验证阶段

## 三、三大策略

### 3.1 双均线趋势跟踪

```
逻辑：MA(10)上穿MA(30) → BUY；MA(10)下穿MA(30) → SELL
参数：fast=10, slow=30
适合：🟢 强势趋势市
```

### 3.2 动量突破

```
逻辑：收盘突破过去20日最高价×1.02 → BUY；跌破20日最低价 → SELL
参数：lookback=20, buffer=2%
适合：🟢 强势趋势市
```

### 3.3 均值回归（布林带）

```
逻辑：收盘<布林下轨(20,2) 且 MA60上升 → BUY；收盘>布林上轨 → SELL
注意：有MA60趋势过滤，防止下跌趋势中反复抄底
适合：🟡 震荡市
```

## 四、三道防线系统

防线执行顺序至关重要——先排除硬性障碍，再做时机判断。

### 第一道：大盘择时

```
判断沪深300 vs MA20/MA60：
  🟢 强势：指数>MA20 且 MA20>MA60       → 仓位系数 1.0
  🟡 震荡：指数>MA20 但 MA20<MA60       → 仓位系数 0.3
  🟠 弱势：指数<MA20                    → 仓位系数 0.0（禁买）
  🔴 极弱：指数<MA60 且连续10日下跌       → 仓位系数 0.0（建议清仓）

策略权重自动调节：
  强势 → 趋势策略（双均线/动量突破）增强+20%，回归策略降权
  震荡 → 回归策略增强+20%，趋势策略降权至30%
```

### 第二道：基础风控

```
排除：ST股票、涨停股、跌停股、停牌股、新股(<60天)
限制：股价>上限（小资金买不起1手）、日成交额<2000万（流动性差）
```

### 第三道：仓位控制

```
按本金自动分档：
  1-2万：单票≤50%，止损-3%，最多2只
  2-5万：单票≤30%，止损-5%，最多4只
  5-10万：单票≤20%，止损-5%，最多6只
  10万+：单票≤15%，止损-5%，最多10只
```

## 五、市场日报

### 5.1 四层分析结构

```
📊 主要指数    沪深300/上证/深证/创业板 涨跌幅+5日涨跌
📈 市场广度    涨跌家数、平均涨跌、成交额
🟢/🟡/🟠/🔴  大盘状态判断

🏭 行业板块    同花顺90个行业 Top5/Bottom5
🔍 重点关注    半导体/光伏/电池/白酒/银行/电力等
💰 股价分层    <10/10-30/30-50/50-100/>100 各档涨跌
🔄 风格判断    价值vs成长、大盘vs小盘

🔍 板块内个股  关键行业领涨领跌股
🔥 异动个股    沪深300涨跌幅前5/后5
🎯 信号复盘    昨日信号的实际涨跌（需积累几天数据）
```

### 5.2 数据来源一致性

日报所有组件强制使用同一天的数据：
- 从 DB 查询最新日期 → 统一传入所有分析函数
- 指数数据按统一日期查询 AKShare，找不到时回退最新
- 数据非今日时在日报标题下方 ⚠️ 标注
- 数据异常时显示告警而非 0 值

## 六、数据质量保障

### 五层保障体系

```
第一层：下载重试      300只×3次重试(退避0.5s/1.0s) + 第二轮补漏(间隔1.0s)
第二层：pct_change修复 新浪源不提供涨跌幅 → SQL从DB前一天收盘价反算
第三层：质量检查       日期=今天？ 股票数≥280？ pct_change无NULL？ 行业数≥80？
第四层：日报一致性     所有分析组件统一data_date
第五层：日报兜底       report.py独立执行fix+verify，不依赖run.py
```

## 八、开发流程

### 8.1 日常开发

```bash
# 本地
source venv/bin/activate
python run.py --no-update     # 快速测试信号
python analysis/report.py     # 快速测试日报

# 部署到服务器
git add -A && git commit -m "描述" && git push
# 然后 SSH 到服务器: cd /root/Liang-Hua && git pull
```

### 8.2 策略迭代规范

```
1. 改参数 → 本地 run.py --no-update 观察信号
2. 跑回测 → python backtest/simple_backtest.py 对比指标
3. 确认无误 → git commit -m "param: 说明" + push
4. 服务器 git pull
5. 观察几天信号质量

出问题时: git log --oneline -5 → git revert <commit> → push
```

### 8.3 添加新策略

```
1. 创建 strategies/new_strat.py
   继承 BaseStrategy，实现 calculate() 和 get_signal()
2. 在 engine/runner.py STRATEGY_REGISTRY 中注册
3. 在 config/settings.py ENABLED_STRATEGIES 中启用
4. 本地测试：python run.py --no-update
5. 跑回测验证
6. 部署
```

| 问题 | 教训 |
