## 三、逐源详细评估

### 3.1 AKShare（当前使用 ⭐）

| 维度 | 评估 |
|------|------|
| **费用** | 完全免费 |
| **安装** | `pip install akshare`，零配置 |
| **日线行情** | ✅ 新浪源稳定，沪深300全覆盖 |
| **财务数据** | ⚠️ 有接口但字段不全，偶尔缺季度数据 |
| **行业数据** | ✅ 同花顺 90 行业，日频 |
| **更新速度** | 收盘后 1-3 小时 |
| **文档质量** | ⭐⭐ 中文文档，但部分接口可能过时 |
| **稳定性** | ⭐⭐⭐ 偶尔抽风，但重试机制能兜底 |

**财务数据获取**：
```python
# 已有接口
ak.stock_financial_abstract(symbol="600519")  # 财务摘要
ak.stock_a_lg_indicator(symbol="600519")       # 成长指标（PE/PB/ROE等）
```

**结论**：日线+行业数据继续用 AKShare。财务数据可用但需验证完整性。

---

### 3.2 Tushare（推荐作为财务数据主力 ⭐⭐⭐⭐）

| 维度 | 评估 |
|------|------|
| **费用** | 基础免费（需注册），高级功能需积分（120 元/年可满足大部分需求） |
| **安装** | `pip install tushare`，注册获取 token |
| **日线行情** | ✅ 稳定，覆盖全A股 |
| **财务数据** | ✅✅ 最全的免费财务数据源：PE/PB/ROE/营收/利润/现金流/负债率 |
| **行业数据** | ✅ 申万行业分类 + 行业指数 |
| **更新速度** | 收盘后 2-4 小时 |
| **文档质量** | ⭐⭐⭐⭐ 完善的中文文档，示例代码丰富 |
| **稳定性** | ⭐⭐⭐⭐ 商业运营，比纯开源稳定 |

**关键接口**：
```python
import tushare as ts
ts.set_token('your_token')
pro = ts.pro_api()

# 日线（备用）
pro.daily(ts_code='000001.SZ', start_date='20190101')

# 财务数据（核心价值）
pro.fina_indicator(ts_code='000001.SZ')            # 财务指标：ROE/ROA/毛利率
pro.daily_basic(ts_code='', trade_date='20260601') # 每日指标：PE/PB/换手率
pro.income(ts_code='000001.SZ')                     # 利润表
pro.balancesheet(ts_code='000001.SZ')              # 资产负债表
```

**费用明细**：
```
注册即送 200 积分 → 足够日常使用
高级接口需 ≥ 2000 积分 → 捐赠 120 元/年
推荐方案：先用免费额度，不够再升级
```

**结论**：财务数据的最佳来源。120 元/年可解锁全量数据，物超所值。

### 3.3 Baostock（🔑 关键发现——推荐作为财务数据主力 ⭐⭐⭐⭐）

| 维度 | 评估 |
|------|------|
| **费用** | 完全免费，**无需注册、无需 Token、无调用限制** |
| **安装** | `pip install baostock` |
| **日线行情** | ✅ 全A股，1990年至今，前复权/后复权/不复权 |
| **日线自带估值** | ✅✅ **K线接口直接返回 PE-TTM、PB-MRQ、PS-TTM、PCF-TTM** |
| **ROE 数据** | ✅ `query_profit_data()` 直接返回 roeAvg |
| **季频财务** | ✅ 利润表+资产负债表+现金流量表+杜邦分析+成长能力 |
| **行业数据** | ❌ 无行业分类 |
| **更新速度** | 交易日 17:30 日K线入库，次自然日 1:30 财务数据入库 |
| **文档质量** | ⭐⭐ 简单，但接口清晰 |
| **稳定性** | ⭐⭐⭐⭐ 数据库驱动（非爬虫），稳定不封 IP |
| **速度** | ⭐⭐⭐⭐ 遍历全市场 < 9 分钟 |

**🔑 最重要的发现**：

```python
# Baostock K线接口直接返回 PE/PB——不需要单独下载财务数据！
rs = bs.query_history_k_data_plus(
    "sh.600000",
    "date,close,peTTM,pbMRQ,psTTM,pcfNcfTTM",  # ← PE/PB/PS/PCF 直接在内
    start_date='2019-01-01', end_date='2026-06-01',
    frequency="d", adjustflag="2"
)
```

这意味着我们的多因子策略只需要一个 API 调用就能同时拿到**价格和估值数据**。300 只股票 × 7 年 = 几百万条 PE/PB 数据，无需逐个下载财务报告。

**和 AKShare 的对比**：
| | AKShare Sina 财务接口 | Baostock K线接口 |
|------|:--:|:--:|
| PE/PB | ❌ 需单独下载，每次一只，极慢 | ✅ 和K线同一接口返回 |
| 单只耗时 | 5-15 秒 | < 1 秒 |
| 300只总耗时 | 数小时（实测卡死） | 预计 < 10 分钟 |
| 稳定性 | ⚠️ 爬虫，可能被 ban | ✅ 数据库驱动 |

**结论**：Baostock 是我们缺失的拼图。日线继续用 AKShare，财务因子用 Baostock K线接口一次性解决。

## 五、推荐方案（更新——基于实测教训）

### 🔑 关键发现

经过 2026-06-14 实测：

```
Tushare：daily_basic 接口需要 ≥ 120 积分（付费），免费注册无法获取 PE/PB
AKShare Sina 财务接口：每只股票 5-15 秒，300 只需数小时，实测卡死在 48 分钟
Baostock：K线接口直接返回 PE/PB/PS/PCF——和价格数据一次请求同时到手！
```

### 最终数据架构

```
本地 SQLite（唯一数据源）
│
├── 日线行情 + PE/PB   ← Baostock K线接口（价格和估值一起返回）
├── ROE/财务指标       ← Baostock query_profit_data()
├── 行业数据           ← AKShare 同花顺（已验证稳定）
├── 指数数据           ← AKShare
└── 股票信息           ← AKShare 中证指数

总成本：0 元（全部免费，无需注册）
```

### 为什么 Baostock 比 Tushare 更适合我们

| | Tushare | Baostock |
|------|:--:|:--:|
| PE/PB 获取 | 需要积分（≥120分=120元/年） | 免费，K线接口自带 |
| 调用限制 | ~200次/日（免费版） | 无限制 |
| 注册 | 需实名认证 + token | **无需注册** |
| 数据稳定性 | 社区采集 | 数据库驱动 |
| 适用场景 | 专业机构 | **个人量化** ✅ |

### 接入计划（简化版）

```
Phase 1：Baostock PE/PB 数据（15 分钟搞定）
  1. pip install baostock
  2. 遍历 300 只股票，用 K线接口一次性拉取 PE/PB + 价格
  3. 预计耗时：< 10 分钟（vs AKShare 数小时）

Phase 2：Baostock ROE 数据（可选）
  1. query_profit_data() 获取季度 ROE
  2. 预计耗时：< 5 分钟

Phase 3：数据质量验证
  1. PE/PB 异常值过滤（<0 或 >500）
  2. 覆盖度检查（≥ 280 只）
```

## 六、数据质量风险（更新）

| 风险 | 说明 | 应对 |
|------|------|------|
| 财务数据滞后 | 季报发布有 1-2 个月延迟 | 使用「最近可用的数据」，不用报告期当日期 |
| PE/PB 异常值 | 微利股 PE 可上万 | 过滤 PE<0、PE>500 |
| Baostock 无实时数据 | 日 K 线 17:30 更新 | 日频策略够用；实时用 AKShare 补充 |
| Baostock 无行业分类 | 需额外处理 | 用 AKShare 同花顺补充 |
| API 不稳定 | Baostock 数据库驱动，稳定 | 比爬虫型 API 可靠得多 |

## 七、网络环境问题（新增）

### 我们的特殊约束

```
Mac 系统代理（Clash）→ 拦截部分 API 请求
  ✅ AKShare 新浪源 → 正常（日线数据稳定下载）
  ✅ Baostock → 待测试（数据库驱动，应不受代理影响）
  ❌ AKShare 东方财富源 → 被拦（stock_zh_a_spot_em 等全灭）
  ❌ AKShare Sina 财务接口 → 极慢（可能走代理绕路）
```

### 最佳实践

```
1. 下载前清除代理环境变量（run.py 已实现）
2. 优先用非东方财富源的数据接口
3. Baostock 端口 80/443 标准 HTTP，理论上不受代理影响
4. 网络实在不行 → 切换到腾讯云服务器上跑下载（服务器无代理）
```
