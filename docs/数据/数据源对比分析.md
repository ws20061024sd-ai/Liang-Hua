## 三、逐源详细评估

### 3.1 AKShare（当前使用 ⭐）

| 维度 | 评估 |
|------|------|
| **费用** | 完全免费 |
| **安装** | `pip install akshare`，零配置 |
| **日线行情** | ✅ 新浪源稳定，沪深300全覆盖 |
| **财务数据** | ⚠️ 有接口但字段不全，偶尔缺季度数据 |
| **行业数据** | ✅ 同花顺 90 行业，日频 |
| **更新速度** | 收盘后 1-3 小时 |
| **文档质量** | ⭐⭐ 中文文档，但部分接口可能过时 |
| **稳定性** | ⭐⭐⭐ 偶尔抽风，但重试机制能兜底 |

**财务数据获取**：
```python
# 已有接口
ak.stock_financial_abstract(symbol="600519")  # 财务摘要
ak.stock_a_lg_indicator(symbol="600519")       # 成长指标（PE/PB/ROE等）
```

**结论**：日线+行业数据继续用 AKShare。财务数据可用但需验证完整性。

---

### 3.2 Tushare（推荐作为财务数据主力 ⭐⭐⭐⭐）

| 维度 | 评估 |
|------|------|
| **费用** | 基础免费（需注册），高级功能需积分（120 元/年可满足大部分需求） |
| **安装** | `pip install tushare`，注册获取 token |
| **日线行情** | ✅ 稳定，覆盖全A股 |
| **财务数据** | ✅✅ 最全的免费财务数据源：PE/PB/ROE/营收/利润/现金流/负债率 |
| **行业数据** | ✅ 申万行业分类 + 行业指数 |
| **更新速度** | 收盘后 2-4 小时 |
| **文档质量** | ⭐⭐⭐⭐ 完善的中文文档，示例代码丰富 |
| **稳定性** | ⭐⭐⭐⭐ 商业运营，比纯开源稳定 |

**关键接口**：
```python
import tushare as ts
ts.set_token('your_token')
pro = ts.pro_api()

# 日线（备用）
pro.daily(ts_code='000001.SZ', start_date='20190101')

# 财务数据（核心价值）
pro.fina_indicator(ts_code='000001.SZ')            # 财务指标：ROE/ROA/毛利率
pro.daily_basic(ts_code='', trade_date='20260601') # 每日指标：PE/PB/换手率
pro.income(ts_code='000001.SZ')                     # 利润表
pro.balancesheet(ts_code='000001.SZ')              # 资产负债表
```

**费用明细**：
```
注册即送 200 积分 → 足够日常使用
高级接口需 ≥ 2000 积分 → 捐赠 120 元/年
推荐方案：先用免费额度，不够再升级
```

**结论**：财务数据的最佳来源。120 元/年可解锁全量数据，物超所值。

### 3.3 Baostock（免费备选 ⭐⭐⭐）

| 维度 | 评估 |
|------|------|
| **费用** | 完全免费，无需注册 |
| **安装** | `pip install baostock` |
| **日线行情** | ✅ 全A股，前复权/后复权 |
| **财务数据** | ✅ 利润表+资产负债表+现金流量表 |
| **行业数据** | ❌ 无行业分类 |
| **更新速度** | 收盘后 4-6 小时（偏慢） |
| **文档质量** | ⭐⭐ 简单，接口较少 |
| **稳定性** | ⭐⭐⭐ 稳定但更新慢 |

**结论**：AKShare 挂了时的免费备选。财务数据没有 Tushare 丰富。

### 3.4 聚宽（在线平台 ⭐⭐⭐）

| 维度 | 评估 |
|------|------|
| **费用** | 免费（研究环境），实盘需付费 |
| **使用方式** | 浏览器在线，不提供本地 Python 包 |
| **日线行情** | ✅ 10+ 年，质量高 |
| **财务数据** | ✅ PE/PB/ROE 等通过 `get_fundamentals()` |
| **因子库** | ✅ 200+ 个预计算因子，直接使用 |
| **局限性** | 只能在聚宽平台内使用，无法导出到本地 |

**适用场景**：快速验证想法（30 秒出回测结果），不依赖本地数据。

**结论**：作为本地回测的补充验证工具，不替代本地数据库。

## 五、推荐方案

### 数据架构

```
本地 SQLite（唯一数据源）
│
├── 日线行情          ← AKShare（主） Tushare（备）
├── 股票信息          ← AKShare
├── 财务数据（新增）   ← Tushare（主） AKShare（备）
├── 行业数据          ← AKShare 同花顺
└── 指数数据          ← AKShare

外部验证：
  聚宽 ← 快速验证策略想法（不在本地存储）
```

### 接入计划

```
Phase 1：Tushare 财务数据
  1. pip install tushare
  2. 注册 tushare.pro → 获取 token
  3. 写下载脚本 → SQLite 新表 financial_data
  4. 字段：code, date, pe, pb, roe, revenue_yoy, profit_yoy, market_cap

Phase 2：AKShare 兜底
  1. 已有 stock_a_lg_indicator 接口
  2. Tushare 失败时自动切换
  3. 和日线下载一样的双源容错逻辑

Phase 3：数据质量验证
  1. 检查 PE/PB 异常值（负值、超大值）
  2. 财务数据发布日期 ≠ 报告期（用最近可用数据）
  3. 季报/中报/年报的发布延迟
```

### 费用

```
AKShare：免费
Tushare：免费（基础额度够用，高级功能 120 元/年）
Baostock：免费（备用）
总计：0 元（初期），120 元/年（如需高级接口）
```

## 六、数据质量风险

| 风险 | 说明 | 应对 |
|------|------|------|
| 财务数据滞后 | 季报发布有 1-2 个月延迟 | 使用「最近可用的数据」，不用报告期当日期 |
| 财务数据缺失 | 部分新股/小票数据不全 | 缺失数据标记，不参与当月排名 |
| PE/PB 异常值 | 微利股 PE 可上万 | 过滤 PE<0、PE>500 |
| 收益质量差异 | 非经常性损益 vs 经常性 | 优先用扣非净利润 |
| API 不稳定 | Tushare 偶发限流 | 双源容错：Tushare→AKShare→跳过 |

---

## 七、结论

> **日线+行业继续用 AKShare。新增 Tushare 获取财务数据（PE/PB/ROE/增速）。总成本 0-120 元/年。**
