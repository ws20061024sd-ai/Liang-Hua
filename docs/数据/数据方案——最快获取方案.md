# 数据方案——最快获取方案

> 编写：2026-06-14 | 基于多方调研 + 实测对比
> 问题：Baostock 单线程 300 只需 7-10 分钟，Mac 代理环境下更慢

---

## 一、当前瓶颈

```
Baostock 单线程：300 只 × 1.5s = 7.5 分钟（理想）
实际：Mac 代理环境导致部分请求卡死，实测多次失败
服务器：待验证（正在跑）
```

## 二、调研结论：三套方案

### 方案 A：Baostock + 多线程（推荐，0 元）

**改进点**：用 `ThreadPoolExecutor` 8 线程并行下载

```python
from concurrent.futures import ThreadPoolExecutor

def fetch_one(code):
    rs = bs.query_history_k_data_plus(...)
    return rs.get_data()

with ThreadPoolExecutor(max_workers=8) as pool:
    results = list(pool.map(fetch_one, codes))
```

**预计提速**：300 只从 7.5 分钟 → **< 2 分钟**

**优点**：零额外依赖，Baostock 已经接入完毕，只改几行代码
**缺点**：仍然受 Mac 代理环境影响

---

### 方案 B：mootdx（通达信本地接口，0 元，最快 ⭐⭐⭐⭐⭐）

**核心发现**：通达信客户端自带本地数据文件，Python 库 `mootdx` 直接读取——**不需要走网络**。

```python
from mootdx.quotes import Quotes
client = Quotes.factory(market='std')
# 读取本地日线数据（毫秒级）
df = client.bars(symbol='600519', frequency=9, offset=0)
# 读取财务数据
df_finance = client.finance(symbol='600519')
```

| 维度 | mootdx | Baostock |
|------|:--:|:--:|
| **速度** | 毫秒级（读本地文件） | 1-2秒/只（网络请求） |
| **PE/PB** | ✅ 日线自带 | ✅ K线自带 |
| **财务数据** | ✅ 本地文件 | ✅ 网络请求 |
| **依赖** | 需安装通达信客户端（免费） | 无需安装 |
| **网络** | 不需要网络 | 需要 |
| **300只耗时** | **< 10 秒** | 2-8 分钟 |
| **稳定性** | 极高（纯本地） | 中（受网络影响） |

**唯一门槛**：需要先安装通达信客户端，让它下载一次全市场数据。之后的日常使用，`mootdx` 直接读本地文件。

## 五、多线程实现要点

改 `financial_downloader.py` 的 `download_financial_data()` 函数：

1. 把单只股票的下载逻辑抽成独立函数 `_download_one_stock(code)`
2. 用 `ThreadPoolExecutor(max_workers=8)` 并行调用
3. 保持原有的 3 次重试逻辑在单只函数内
4. 保持 Baostock QPS ≈ 20 的限制（8 线程每只 1.5s，实际 QPS ≈ 5，远低于上限）

## 六、结论

> **Baostock 本身不慢——是我们的单线程实现慢。改成多线程即可 2 分钟内完成。终极方案是 mootdx（纯本地，秒级），但需要先装通达信。**
