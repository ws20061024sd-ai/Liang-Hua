## 2. A股回测框架：哪个适合你？

### 2.1 Backtrader ⭐ ~10k

```
GitHub: github.com/mementum/backtrader

一句话：量化入门的事实标准，文档和中文资料最丰富。

优点：
  ✅ 事件驱动架构，逻辑清晰
  ✅ 内置 40+ 技术指标（MA、MACD、RSI、布林带...）
  ✅ 中文教程极多，B站/公众号随便搜
  ✅ 完善的佣金、滑点、仓位管理
  ✅ 社区活跃，Bug 有人管

缺点：
  ❌ 纯 Python，数据量大时慢
  ❌ 不原生支持 A 股 T+1 和涨跌停（需要自己写）
  ❌ 原作者维护不积极（社区 fork 在维护）

适合你吗？ ✅ 非常适合。
  - 你的自建系统目前没有回测引擎，正好可以用 Backtrader 补上
  - 把 AKShare 数据 feed 进去，策略逻辑搬到 Backtrader 的 Strategy 类里
  - 一周内就能跑出第一个回测报告

学习成本：⭐⭐（2/5，非常低）
推荐指数：⭐⭐⭐⭐⭐（5/5，入门首选）
```

### 2.2 Hikyuu ⭐ ~2k

```
GitHub: github.com/fasiondog/hikyuu

一句话：C++内核 + Python 外壳，全A股回测快到飞起。

优点：
  ✅ C++ 核心，性能极强（全A股日线 MA 计算仅 166ms）
  ✅ 组件化设计：市场环境判断、信号指示器、止损止盈各自独立
  ✅ 可复用策略组件，逐步积累自己的策略资产库
  ✅ 内置 HDF5/MySQL 本地存储
  ✅ 有完整的网站文档：hikyuu.org

缺点：
  ❌ 学习曲线比 Backtrader 陡
  ❌ 社区比 Backtrader 小，遇到问题搜不到答案
  ❌ 安装稍复杂（有 C++ 依赖）

适合你吗？ 🟡 现在不太适合，但值得关注。
  - 当你策略多了、回测太慢时，Hikyuu 是很好的升级选择
  - 现在先用 Backtrader，等熟练了再切过来

学习成本：⭐⭐⭐⭐（4/5，较高）
推荐指数：⭐⭐⭐（3/5，进阶推荐）
```

### 2.3 VectorBT ⭐ ~4k

```
GitHub: github.com/polakowo/vectorbt

一句话：向量化回测，比 Backtrader 快 10-100 倍。

优点：
  ✅ 极快的参数扫描（几千组参数秒出结果）
  ✅ 天然支持热力图可视化
  ✅ 适合做大规模参数优化

缺点：
  ❌ 向量化回测逻辑不同于事件驱动，需要换思维
  ❌ 复杂的动态止损不好实现

适合你吗？ 🟡 优化参数时用。
  - 不适合日常回测（逻辑不够灵活）
  - 当你需要测试 "MA5/10/20/30/60 × MA10/20/30/60/120 哪种组合最好" 时用它

学习成本：⭐⭐⭐（3/5）
推荐指数：⭐⭐⭐⭐（4/5，特定场景极强）
```

### 2.4 Zipline / Zipline-Reloaded

```
GitHub: github.com/quantopian/zipline (原版)
       github.com/stefan-jansen/zipline-reloaded (社区维护版)

一句话：Quantopian 遗产，事件驱动回测的教科书。

优点：
  ✅ 架构设计精良，行业标准级
  ✅ 时间戳处理极好（天然防未来函数）
  ✅ 和 pyfolio（绩效分析）+ Alphalens（因子分析）完美配合

缺点：
  ❌ Quantopian 已倒闭，原版停更
  ❌ A股适配需要额外折腾
  ❌ 安装麻烦，依赖老旧

适合你吗？ ❌ 不适合。
  - 已经过时了，不要再学
  - Backtrader 是更好的替代

推荐指数：⭐⭐（2/5，仅学术参考价值）
```

## 3. 全流程量化平台：一站式方案

### 3.1 阿布量化 (ABU) ⭐ ~11k

```
GitHub: github.com/bbfamily/abu

一句话：中文量化教程做得最好的项目，19节 Jupyter Notebook 手把手教。

优点：
  ✅ 19 节 Jupyter Notebook 教程，从零教起
  ✅ 内置 A 股规则（T+1、涨跌停板）
  ✅ 缠论、波浪理论等形态识别
  ✅ 自动衍生 18496 种策略组合
  ✅ 纯 Python，安装简单

缺点：
  ❌ 框架稍显老旧（Python 2/3 过渡期产物）
  ❌ 不太适合直接用于实盘
  ❌ 代码风格有些地方不够 Pythonic

适合你吗？ ✅ 非常适合学习！
  - 不是用来跑实盘的，是用来看教程、理解量化思维的
  - 把 19 节 Notebook 全部跑一遍，你对量化的理解会上一个大台阶
  - 这可能是对你目前最有价值的单个项目

学习成本：⭐⭐（2/5，教程做得太好了）
推荐指数：⭐⭐⭐⭐⭐（5/5，学习首选）
```

### 3.2 FinHack ⭐ ~1k

```
GitHub: github.com/FinHackCN/finhack

一句话：2025年最活跃的中文全流程量化框架。

优点：
  ✅ 2025 年仍在活跃更新
  ✅ 全流程覆盖：数据采集→因子计算→因子挖掘→ML→回测→实盘
  ✅ 支持 A 股/美股/期货/加密货币
  ✅ DuckDB 高性能存储
  ✅ 模块化设计，可以只用其中一部分

缺点：
  ❌ 社区还小，文档不够完善
  ❌ 项目结构复杂，新人容易迷路

适合你吗？ 🟡 可以作为架构参考。
  - 不必直接用，但可以学习它的模块划分、代码组织方式
  - 看它的 README 和目录结构就能学到很多

学习成本：⭐⭐⭐⭐（4/5）
推荐指数：⭐⭐⭐（3/5，架构设计参考）
```

### 3.3 VN.PY ⭐ ~30k

```
GitHub: github.com/vnpy/vnpy

一句话：国内实盘对接能力最强，GitHub Stars 最高的中文量化项目。

优点：
  ✅ 对接 40+ 国内券商/期货公司接口
  ✅ 支持 A 股、期货、期权、黄金 T+D
  ✅ C++ 加速，支持高频交易
  ✅ 2025 年发布 4.0，加入 AI 因子模块
  ✅ 可视化桌面界面（PyQt）

缺点：
  ❌ 回测功能弱（它主要做交易执行，不是回测研究）
  ❌ 安装复杂，依赖多
  ❌ 学习曲线陡峭

适合你吗？ 🟡 现在不适合，但记住它——实盘时你会需要。
  - 你现在是"生成信号→人工下单"的阶段，vnpy 用不上
  - 等你想"策略自动下单"时，vnpy 是最好的选择

学习成本：⭐⭐⭐⭐⭐（5/5，非常高）
推荐指数：⭐⭐⭐⭐（4/5，实盘终极方案）
```

## 4. AI/ML 量化：进阶方向

> ⚠️ 先声明：你现在不需要这些。先把传统策略（均线、动量、因子）搞清楚，AI 是加分项不是必须项。以下列出供你以后参考。

### 4.1 微软 Qlib ⭐ ~11k

```
GitHub: github.com/microsoft/qlib

一句话：微软出的 AI 量化平台，学术研究和工业落地都用它。

优点：
  ✅ 内置 Alpha158 因子集（业界标准）
  ✅ 内置 LSTM、Transformer、GBDT 等预训练模型
  ✅ 数据处理、训练、回测全流程
  ✅ 有 A 股数据下载能力
  ✅ 科研成果多（KDD、AAAI 论文都在用）

缺点：
  ❌ 学习曲线非常陡
  ❌ 需要 ML 基础
  ❌ 对新手不友好（官方文档偏学术）

适合你吗？ 🟡 3-6 个月后可以考虑。
  - 当你熟练了传统策略、理解了因子投资后
  - Qlib 是你进入 AI 量化世界的第一个路口

推荐指数：⭐⭐⭐⭐（4/5，AI 量化第一选择）
```

### 4.2 FinRL ⭐ ~9k

```
GitHub: github.com/AI4Finance-Foundation/FinRL

一句话：用强化学习训练交易智能体。

优点：
  ✅ DQN、PPO、SAC 等主流 RL 算法
  ✅ 有教程和论文支撑
  ✅ 支持多市场

缺点：
  ❌ 强化学习在交易中稳定性差
  ❌ 实盘效果远不如回测

适合你吗？ ❌ 现在完全不适合。
  - 强化学习交易学术上很火，实盘上坑很多
  - 先别碰

推荐指数：⭐⭐（2/5，学术玩具）
```

### 4.3 Stock-Prediction-Models ⭐ ~9k

```
GitHub: github.com/huseinzol05/Stock-Prediction-Models

一句话：一个人的疯狂实验，几百种模型的股票预测尝试。

优点：
  ✅ 模型种类极多（LSTM、GAN、RL、Transformer...）
  ✅ 可以快速看不同模型的效果

缺点：
  ❌ 代码质量参差不齐
  ❌ 没有系统化的框架
  ❌ 更像一个实验记录，不适合直接使用

适合你吗？ 🟡 无聊时翻翻看。
  - 不是用来学的，是用来"开眼界"的
  - 翻翻 readme，看看有哪些思路是你不曾想过的

推荐指数：⭐⭐（2/5，灵感来源）
```
