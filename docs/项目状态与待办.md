### 方案一：Bark（🏆 最推荐）

**原理**

```
run.py → HTTP请求 → Bark服务器 → iPhone通知
                        ↓
              Apple Push Notification Service (APNs)
                        ↓
                   你的iPhone弹出通知
```

**iPhone 端**：App Store 搜 "Bark" 安装，打开即得到一个推送 URL（如 `https://api.day.app/xxxx`）

**代码**：一行搞定

```python
import requests
requests.get(f"https://api.day.app/{YOUR_KEY}/量化信号/今日买入2只，卖出0只")
```

**效果**：

```
┌─────────────────────────┐
│ 📊 量化信号              │
│ 今日买入2只，卖出0只      │
│ 四川路桥 MA10上穿MA30    │
└─────────────────────────┘
```

| 优点 | 缺点 |
|------|------|
| 配置只需 1 分钟 | 仅支持 iPhone |
| 免费、无广告 | 不支持 Markdown 排版 |
| 电池耗电极低（走 APNs） | 消息长度有限制 |
| 开源，可自建服务器 | — |
| ⭐ 最推荐新手 | — |

---

### 方案二：钉钉机器人

**原理**

```
run.py → HTTP请求 → 钉钉服务器 → 钉钉群消息
```

**配置步骤**：

1. 建个钉钉群（只拉自己）
2. 群设置 → 智能群助手 → 添加自定义机器人
3. 复制 Webhook URL

**代码**：约10行

```python
import requests, json
webhook = "https://oapi.dingtalk.com/robot/send?access_token=xxx"
data = {
    "msgtype": "markdown",
    "markdown": {
        "title": "量化交易信号 2026-06-07",
        "text": "## 🟢 买入建议\n- 四川路桥(600039) MA10上穿MA30\n- 江苏银行(600919) MA10上穿MA30\n\n---\n## ⚠️ 持仓关注\n无"
    }
}
requests.post(webhook, json=data)
```

**效果**：钉钉群聊天消息，支持 Markdown 排版

| 优点 | 缺点 |
|------|------|
| 支持 Markdown（排版美观） | 需要安装钉钉 App |
| iPhone/Android 都支持 | 信号信息安全问题（自建群=只有自己看，无碍） |
| 消息长度无限制 | 需要建群、加机器人（多几步） |
| 可发多条，可 @自己 | 推送走钉钉通道，可能有延迟 |

### 方案三：企业微信机器人

和钉钉几乎一样，接口兼容。如果你常用企业微信而非钉钉，选这个。

| 优点 | 缺点 |
|------|------|
| 同上，Markdown 支持好 | 需要企业微信 App |
| 微信生态用户更方便 | 个人号需注册企业（免费） |

### 方案四：Server酱

**原理**：

```
run.py → Server酱 API → 微信公众号 → 微信消息
```

关注 Server酱公众号 → 获取 SendKey → 一行代码发送。

| 优点 | 缺点 |
|------|------|
| 走微信，无需额外 App | 免费版每日限制 5 条 |
| 配置极简 | 排版能力弱 |
| 人人都有微信 | 曾有过不稳定时期 |

```python
import smtplib
# SMTP 发邮件到你的邮箱

| 优点 | 缺点 |

| 零依赖 | 排版差 |

Q: 你现在用什么手机？

→ 首选 Bark（最轻量，1分钟搞定）
→ 备选 钉钉机器人（排版更好，功能更强）
→ 两者可以同时用，不冲突
```

| 如果你想要... | 选 |
|--------------|-----|
| 最省事，看到就行 | **Bark** |
| 排版好看，信号一目了然 | **钉钉机器人** |
| 两个都要 | Bark + 钉钉同时推送 |
