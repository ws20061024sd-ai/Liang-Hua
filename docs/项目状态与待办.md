```
框架层 ✅ 已完工  ──→  策略层 ⏳ 现在开始

数据/风控/推送/服务器      回测验证 → 策略扩展 → 实盘跟单
```

当前只有 1 个双均线策略，且**从未在历史数据上回测过**——不知道它到底靠不靠谱。第二阶段的目标就是回答这个问题。

### 阶段一：回测验证（第一优先级）

**目标**：用历史数据验证双均线策略的真实表现

#### 方案 A：聚宽在线回测（推荐起步）

```
为什么先选聚宽：
  ✅ 零代码搭建——浏览器里点几下就跑
  ✅ 数据齐全——自带沪深300十年数据
  ✅ 报告完整——收益率/夏普/回撤/胜率一键出
  ✅ 社区模板——直接搜"双均线"有现成代码

操作步骤：
  1. joinquant.com 注册
  2. 复制双均线策略模板
  3. 设置参数：MA10/MA30，沪深300成分股
  4. 回测区间：2020-01-01 到 2026-06-01
  5. 跑回测，看报告

关键指标观察：
  年化收益率 → 有多少？如果<5%不如理财
  最大回撤   → 最惨的时候亏多少？
  夏普比率   → 每单位风险赚多少？>1才算及格
  胜率       → 多少次交易盈利？30-40%正常
  盈亏比     → 平均赚/平均亏 >1.5才行
```

#### 方案 B：本地 Backtrader（聚宽验证通过后）

```
为什么后选本地：
  - 需要写代码搭框架（1-2天）
  - 好处：和实盘信号完全一致的逻辑
  - 好处：可以自定义任何细节
```

#### 回测必须检查的陷阱

```
□ 手续费是否设置（0.03%买卖+0.05%印花税）
□ 滑点是否设置（0.1%）
□ 是否用了未来数据（如"今天用今天才知道的财务数据"）
□ 涨跌停是否处理（涨停买不到、跌停卖不掉）
□ 是否排除了ST和停牌
□ 回测区间是否覆盖至少一轮牛熊
```

### 阶段二：策略扩展（回测通过后）

#### 策略路线图

```
现有：双均线趋势（MA交叉）
         │
    ┌────┴────┐
    ▼         ▼
动量突破    均值回归
(趋势市)    (震荡市)
    │         │
    └────┬────┘
         ▼
    多策略并行
```

#### 策略一：动量突破（强势趋势捕捉）

```
逻辑：
  今日收盘 > 过去 N 日最高收盘价 × (1 + buffer) → BUY
  今日收盘 < 过去 M 日最低收盘价 → SELL

参数：
  lookback = 20天
  buffer = 0.02（突破2%确认）

适合环境：🟢 强势市场（当前大盘择时会自动搭配）
现有框架集成：只需继承 BaseStrategy，注册即可
```

#### 策略二：均值回归（震荡市捕捉）

```
逻辑：
  收盘价 < 布林带下轨(20,2) → BUY（超卖）
  收盘价 > 布林带上轨(20,2) → SELL（超买）
  收盘价回到中轨 → 平仓

注意：
  - 只在🟡震荡市启用（大盘择时提供信号）
  - 需要趋势过滤：MA60向下时不做多
```

#### 多策略信号汇总（需要开发）

```
策略A: BUY 四川路桥, 中远海控
策略B: BUY 四川路桥, 京东方A
策略C: SELL 四川路桥
      ↓
汇总规则：
  - 同股票同方向多策略确认 → 增强置信度
  - 同股票买卖冲突 → 标记冲突，人工裁决
  - 按信号强度排序
```

## 📋 待实施：板块趋势与轮动分析

### 需求

```
当前日报已有：行业板块当天涨跌排名（同花顺API）
缺少：
  1. 板块趋势 —— 过去5/10天板块涨跌走势
  2. 板块轮动 —— 哪个板块在走强/走弱，资金在往哪流
  3. 板块内个股 —— 关键行业谁涨最猛、谁跌最惨
  4. 板块比较 —— 不同行业横向对比（估值/资金/动量）
```

### 方案设计

#### 数据来源

```
当前：同花顺 stock_board_industry_summary_ths() ← 只有当天数据

需要：每天缓存板块数据到 SQLite，积累历史才能算趋势
  sector_history 表：
    date | name | pct_change | up_count | down_count | net_flow
```

#### 分析维度

```
1. 板块趋势（缓存5天数据后可用）
   ├── 5日动量排名 —— 哪些板块最近涨最多
   ├── 资金流向趋势 —— 净流入/流出变化
   └── 持续性 —— 连续上涨天数

2. 板块轮动（缓存10天+数据后可用）
   ├── 新热点 —— 今日涨但之前弱的板块（可能启动）
   ├── 退潮信号 —— 今日跌但之前强的板块（可能见顶）
   └── 轮动方向 —— 从哪个板块流出、流入哪个板块

3. 板块内个股（当天数据即可）
   ├── 关键行业（半导体/光伏/白酒/银行/电力）的涨跌股
   ├── 行业内领涨/领跌
   └── 行业内部分化程度

4. 板块比较
   ├── 估值对比（需要PE/PB数据，后续）
   ├── 资金流入对比
   └── 涨跌家数比
```

#### 日报格式预览

```markdown
### 🏭 行业板块

**今日Top5**：
半导体 +6.38% | 通信设备 +3.3% | 电池 +3.0%

**5日动量最强** 📈：
半导体 +12% | 光伏 +8% | 电池 +6%

**5日动量最弱** 📉：
煤炭 -5% | 白酒 -3% | 银行 -1%

**资金流入Top3** 💰：
半导体 +119亿 | 通信设备 +98亿 | 电池 +34亿

### 🔄 板块轮动
🟢 新启动：消费电子（前期弱，今日放量涨）
🔴 在退潮：煤炭（前期强，今日转跌）

### 🔍 重点行业内个股
**半导体** 173涨/6跌 平均+6.4%
  🟢 沪硅产业 +20% | 晶盛机电 +16%
  🔴 华润微 -0.1%

**白酒** 2涨/18跌 平均-2.5%
  🟢 贵州茅台 +0.3% | 五粮液 -0.1%
  🔴 泸州老窖 -3.2% | 古井贡酒 -3.5%
```

### 实施步骤

```
第一步：建 sector_history 表 + 每日缓存（1个函数，10分钟）
第二步：5日趋势计算（基于缓存数据，1个函数）
第三步：轮动检测（新热点/退潮信号，1个函数）
第四步：板块内个股分析（关键词匹配，1个函数）
第五步：集成到日报 + 钉钉推送格式
第六步：积累10天+数据后，提取轮动规律
```

### 数据积累周期

```
第二天：可以算1日对比
第五天：可以算5日动量
第十天：可以算轮动信号
一个月：可以算板块持续性统计
```

### 风险与局限

```
✓ 同花顺API 90个行业，覆盖面广
✗ 首次上线时没有历史数据，需积累几天
✗ API偶尔不稳定 → 需要本地缓存兜底（已实现 industry.py 关键词分类）
✗ 板块内个股用名称关键词匹配，不够精确
```
